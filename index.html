BEST PRACTICES

Password Security Guidelines

📏

Minimum 16 Characters

Each additional character exponentially increases the search space. 16+ chars with mixed types is virtually uncrackable with modern hardware.

🎲

True Randomness

Use a password manager to generate cryptographically random passwords. Avoid personal info, dictionary words, or predictable substitutions like @ for a.

🔄

Unique Per Account

Never reuse passwords. If one site is breached, attackers use credential stuffing to access all accounts with the same password.

🛡

Enable 2FA Always

Two-factor authentication blocks 99.9% of automated attacks even if your password is compromised. Use TOTP apps, not SMS when possible.

🔐

Use a Password Manager

Bitwarden, 1Password, or KeePassXC can generate, store, and autofill complex passwords securely so you only need to remember one master password.

🚫

Avoid Common Patterns

Password123!, Summer2024, qwerty, and keyboard walks (zxcvbn) are the first guesses in any dictionary or rule-based attack.

We detected suspicious activity. You MUST verify your identity immediately or your account will be permanently deleted.

Click here: http://paypa1-secure-verify.tk/login

⚠ Fake domain · "1" instead of "l" · .tk TLD · Extreme urgency · HTTP not HTTPS
🔤

Lookalike Domains

paypa1.com vs paypal.com — attackers swap letters with similar-looking characters.

⏰

False Urgency

"Act NOW or your account will be deleted!" — creates panic to bypass rational thinking.

🌐

Suspicious TLDs

.tk, .ml, .xyz, .top — free domains commonly used in phishing campaigns.

🔒

No HTTPS

Legitimate sites use HTTPS. HTTP means data is sent unencrypted and can be intercepted.

📎

Unexpected Attachments

.exe, .zip, .docm files from unknown senders often contain malware or ransomware.

🎭

Impersonation

Attackers pose as banks, tech companies, or government agencies to gain trust.
